# OctoAcme Process Improvement Checklists

This document provides consolidated checklists for the key project management stages in OctoAcme. Use it as a lightweight reference to ensure consistent execution and to close common process gaps.

## Initiation Checklist
- [ ] Project One-pager completed and reviewed by Product Lead
- [ ] Stakeholder communication plan drafted and shared
- [ ] Success metrics, scope, and timeline validated
- [ ] Roles and responsibilities assigned
- [ ] Initial risk list and mitigation concepts documented
- [ ] Decision made to move into planning

## Planning Checklist
- [ ] Kickoff held with stakeholders and delivery team
- [ ] Backlog prioritized and estimated
- [ ] Release timeline and milestones agreed
- [ ] Definition of Done documented
- [ ] Initial test plan / QA approach drafted
- [ ] Dependency owners identified and handoff points agreed
- [ ] Product, Project, and Delivery Leads aligned

## Execution Checklist
- [ ] Daily standups capture progress, blockers, and dependencies
- [ ] Weekly delivery sync includes risk and metrics review
- [ ] Quality and release criteria are reviewed ahead of milestones
- [ ] Team follows branching, PR, and CI standards
- [ ] Process gaps and action items are documented for retrospectives
- [ ] Stakeholder communications are updated in the status artifact

## Release Checklist
- [ ] Release readiness reviewed by Project, Product, QA, and Delivery Leads
- [ ] Acceptance criteria complete and tested
- [ ] Deployment plan and rollback plan confirmed
- [ ] Release communications and support arrangements finalized
- [ ] Staging and production smoke tests executed
- [ ] Post-release review and follow-up actions scheduled

## Risk & Communication Checklist
- [ ] Risk register reviewed regularly and kept current
- [ ] New risks assigned owners and mitigations
- [ ] Communication plan covers all stakeholder groups
- [ ] Weekly status updates include progress, risks, and decisions
- [ ] Escalation paths and owners are clearly defined

## Retrospective & Improvement Checklist
- [ ] Retrospective session completed after each milestone or release
- [ ] Action items captured with owners, due dates, and success criteria
- [ ] Improvement items are added to backlog or project board
- [ ] Prior retrospective actions are reviewed for completion
- [ ] Process improvements are shared with the wider team
